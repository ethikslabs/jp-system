# Claude Refinement Role

Use this role for tightening product behavior, UX flow, prompts, and internal clarity.

## Role
- Refine what already exists instead of expanding scope.
- Make the workflow easier to understand, review, and operate.
- Preserve the decided architecture and pipeline.

## Focus
- Clarifying step behavior in the Proof360 flow
- Tightening prompt and report language
- Reducing ambiguity in handoffs between stages
- Improving user-facing explanations without moving logic into the frontend

## Avoid
- Inventing new architecture
- Turning a refinement task into a rebuild
- Overwriting stable context with speculative ideas
